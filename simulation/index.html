Simulation and the Emerging "Digital Twin" Concept

Machine Simulation Through Ignition

I have made a career for myself on creating detailed machine simulations using the Ignition platform:

Simulations are invaluable for flushing out control system design errors, stepping through complicated sequences, and polishing operator interfaces.

Here are some advantages of using Ignition as a simulation platform:

  • Design and develop simulation interface using standard Ignition Perspective/Vision features
  • Communicate to variety of PLCs via built-in drivers
  • Use of direct OPC reads/writes + SFCs for round-trip communications as fast as 50ms
  • Use visual templates and UDTs for ease of creation

Pylogix and Real-time 3D Simulation

pylogix is an open source software allowing reading/writing tag values using the CIP protocol in Python.

There is an experimental branch of pylogix which allows Class 1 (UDP) CIP comms to a generic ETHERNET-MODULE in Rockwell PLCs.

Real-time, 3D simulation of systems is possible (for Rockwell PLCs) with the following technologies:

  • Pylogix Class 1 CIP comms, allowing round trip simulation communication times as low as 10ms
  • Simulation/process logic developed in Python
  • Use of the Godot 3D Game Engine for developing physical simulations, with the Jolt Physics Engine

Python code can be integrated with the Godot Engine in a couple of ways:

  • Through a plugin allowing for a python GDNative plugin (not quite working for Godot 4 yet)
  • Through a local UDP server/client where the game engine emits physics collisions to the Python backend in order to communicate with a PLC
